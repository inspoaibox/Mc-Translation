# Token 限制优化说明

## 问题背景

你指出了一个关键问题：**不同语言对的翻译长度比例差异很大**。

原先的实现使用固定的倍数（小模型 1.5x，标准模型 1.8x），这忽略了语言特性：

### 实际翻译长度变化

| 语言对 | 长度变化 | 原因 |
|--------|---------|------|
| 英文 → 中文 | **缩短 30-50%** | 英文单词变成汉字 |
| 中文 → 英文 | **增长 50-100%** | 汉字扩展为英文单词 |
| 日文 → 英文 | **增长 80-120%** | 假名/汉字 → 英文单词 |
| 英文 → 德文 | **增长 20-30%** | 德语复合词更长 |

### 原有问题

1. **英译中浪费资源**：
   - 输入 100 tokens，实际只需 70 tokens
   - 但分配了 150 tokens (1.5x)，浪费 80 tokens

2. **中译英内容截断**：
   - 输入 100 tokens，实际需要 200 tokens
   - 但只分配了 150 tokens (1.5x)，**截断 50 tokens！**

3. **小模型上限太小**：
   - 256 tokens 对长文本翻译不够

---

## 解决方案

### 1. 语言对比例映射表

```python
lang_pair_ratios = {
    # 英译中/日/韩：通常缩短（英文单词 → 表意文字）
    ("en", "zh"): 0.7,
    ("en", "ja"): 0.8,
    ("en", "ko"): 0.75,
    
    # 中/日/韩译英：通常增长（表意文字 → 英文单词）
    ("zh", "en"): 2.0,
    ("ja", "en"): 2.0,
    ("ko", "en"): 1.8,
    
    # 欧洲语言互译：长度相近
    ("en", "es"): 1.15,
    ("en", "fr"): 1.15,
    ("en", "de"): 1.2,
    ("en", "it"): 1.1,
    ("en", "pt"): 1.15,
    ("en", "ru"): 1.0,
    
    # 其他常见对
    ("zh", "ja"): 1.1,
    ("zh", "ko"): 1.1,
}

# 未知语言对默认 1.5x（保守估计）
ratio = lang_pair_ratios.get((source_lang, target_lang), 1.5)
```

### 2. 动态计算公式

**小模型**：
```python
dynamic_limit = int(input_token_count * ratio) + 32
max_limit = min(512, config.LLM_TRANSLATION_MAX_NEW_TOKENS)
return max(32, min(max_limit, dynamic_limit))
```

**标准模型**：
```python
dynamic_limit = int(input_token_count * ratio * 1.2) + 48
return max(48, min(config.LLM_TRANSLATION_MAX_NEW_TOKENS, dynamic_limit))
```

### 3. 关键改进

| 项目 | 修改前 | 修改后 | 效果 |
|------|--------|--------|------|
| 英译中比例 | 1.5x | **0.7x** | 节省 53% 资源 |
| 中译英比例 | 1.5x | **2.0x** | 避免截断 |
| 小模型上限 | 256 | **512** | 支持更长文本 |
| 标准模型余量 | 0% | **20%** | 更安全 |

## 技术细节

### 为什么需要 buffer？

```python
dynamic_limit = int(input_token_count * ratio) + 32  # 小模型
dynamic_limit = int(input_token_count * ratio * 1.2) + 48  # 标准模型
```

**原因**：
1. **标点符号**：翻译可能添加额外的标点
2. **格式保留**：Markdown、HTML 标签
3. **模型不确定性**：实际比例有波动
4. **安全边界**：避免边缘截断

### 为什么标准模型有 1.2x 额外余量？

标准模型（≥ 1B）能力更强，可能生成：
- 更地道的表达（可能更长）
- 更完整的句子结构
- 更丰富的词汇选择

额外 20% 确保高质量输出不被截断。

### 为什么小模型上限提高到 512？

原先的 256 tokens 对应约：
- **英文**：50-60 个单词（2-3 句话）
- **中文**：80-100 个汉字（1 段话）

提高到 512 tokens 后：
- **英文**：100-120 个单词（5-6 句话）
- **中文**：160-200 个汉字（2-3 段话）

这对商业邮件、产品描述等实际场景更合理。
