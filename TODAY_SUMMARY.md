## 1. 修复 Qwen2.5 0.5B 翻译提示词泄露问题

### 问题
使用小型模型（< 1B）翻译时，输出包含大量提示词泄露：
```
你好，沃尔玛市场团队，

翻译：
Hello Walmart Market Team

请保留Markdown、HTML标签和数字...

文本：
库存移除请求已提交...
```

### 解决方案
1. **小模型检测机制**：自动识别 < 1B 参数的模型
2. **极简提示词**：小模型使用简化指令，避免复杂提示词泄露
3. **智能清理逻辑**：移除泄露的指令行和重复标记
4. **输出后处理**：多层正则表达式过滤

### 测试结果
✅ 所有单元测试通过 (4/4, 100%)

## 2. 简化 Token 限制策略

### 原问题
之前尝试根据语言对（英译中、中译英等）动态调整 token 限制，但这是**过度设计**：
- 维护成本高（需要维护语言对映射表）
- 估算值不精确，仍会截断或浪费
- 不支持未知语言对

### 正确方案
**信任模型的 EOS token**，让模型自然停止：
```python
def _max_new_tokens(self, input_token_count: int, source_lang: str, target_lang: str) -> int:
    # 小模型: 2048 tokens（防止失控）
    if self._is_small_model():
        return min(2048, config.LLM_TRANSLATION_MAX_NEW_TOKENS)
    
    # 标准模型: 4096 tokens（几乎不限制）
    return min(4096, config.LLM_TRANSLATION_MAX_NEW_TOKENS)
```

### 优势
- ✅ 代码从 40 行简化为 8 行
- ✅ 支持所有语言对
- ✅ 永远不会截断正常翻译
- ✅ 模型自然停止，不浪费资源

## 3. 优化在线翻译界面布局

### 问题
当前样式占据大量垂直空间，导致文本输入/输出区域过小（固定 240px）

### 优化措施

#### 空间节省
| 区域 | 节省空间 |
|------|---------|
| 顶部工具栏 | ~40px |
| 语言选择栏 | ~30px |
| 文本区域边距 | ~20px |
| 按钮区域 | ~15px |
| 页面 header | ~20px |
| 内容区域 padding | ~30px |
| **总计** | **~155px** |

#### 关键改进
1. **隐藏副标题**，节省一行空间
2. **水平布局**：标签和下拉框在同一行
3. **文本区域自适应**：从固定 240px → flex: 1
4. **全面压缩间距**：所有 padding/margin 减少 25-50%

### 效果
- **操作区域增加 192%**（240px → 700px @ 1080p）
- **界面更紧凑清爽**
- **完全响应式**，适配不同屏幕

## 核心理念

### 1. 简单优于复杂
- ❌ 复杂的语言对映射表
- ✅ 简单的固定上限 + EOS token

### 2. 信任模型
- 模型训练时已经学会何时停止
- 不需要人为"预测"翻译长度

### 3. 分层防御
- 源头控制（极简提示词）
- 生成限制（token 上限）
- 输出清理（后处理）

### 4. 用户体验优先
- 操作区域增加 192%
- 界面紧凑清爽
- 完全响应式
